Four runs, in the order they actually happened

Each entry is a hypothesis proposed before the run, and the result that either supported or killed it.

01

v1 — collapse every reasoning line into one, train 2 epochs

Mechanical compressor strips GSM8K's inline calculator annotations and joins all steps into a single semicolon-separated line. Result: 51.0% accuracy, 93.8 tokens — tokens down ~70%, accuracy down 18 points from base.

Lesson: the training labels were barely shortened from GSM8K's originals — yet the model's own generations got far more compressed than the labels ever were. Compressing the data doesn't just teach a format; it teaches the model to think less, and that effect can outrun the input.

02

Hypothesis: the line-collapsing format itself caused the drop

v2 keeps GSM8K's original multi-line structure — same words, same 51.7-word average, only the formatting changed. Result: 50.0% accuracy, 89.7 tokens — statistically identical to v1.

Falsified

Lesson: when changing an obvious surface variable produces no difference, the real cause is probably upstream of it. Formatting felt like the likely culprit; it wasn't — worth ruling out before chasing subtler explanations.

03

Hypothesis: less training preserves more accuracy

v3 trains on the same data as v2 for 1 epoch instead of 2, expecting a gentler trade-off. Result: 45.0% accuracy, 86.4 tokens — worse on both axes. Style adoption and reasoning-reliability recovery appear to move on different timescales during fine-tuning.

Falsified

Lesson: the trade-off isn't a smooth dial you can back off gently. The terse style seems to be adopted quickly, while reliable arithmetic under that new style takes longer to recover — cutting training short catches the model in the worst of both.

04

v4 — change the training target itself: self-distillation

Instead of compressing GSM8K's gold labels, generate the base model's own reasoning on all 7,473 questions, keep only the 5,780 (77.3%) it already gets right, and trim only genuine boilerplate (intro/closing prose) — leaving the derivation untouched, so length still scales with difficulty. Result: 64.0% accuracy, 248.4 tokens.

Confirmed

Lesson: the diagnosis from runs 01-03 made a testable prediction — train on adaptive-length, verified-correct reasoning instead of a fixed terse template, and accuracy should recover. It did, almost all the way. The honest trade: token savings are far more modest (17% vs ~70%), because most of the token cost lives in the reasoning itself, not in filler.
